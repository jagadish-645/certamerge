# CertaMerge Self-Dogfood Run Report

## Purpose

This report records the local run where CertaMerge evaluated its own public alpha repository using `.certamerge.yml`.

The policy is intentionally in observe mode. It must produce the same interaction grammar as product users see:

```text
Verdict.
Policy reason.
Missing proof.
Accountable next action.
Change Authorization Record.
```

## Initial Finding

The first local self-gate reached an observe-mode would-block result because the repository had rules that required proof but did not yet carry metadata evidence for the changed surfaces. The missing proof categories were test proof, owner approval, and dependency/package evidence.

That was a good failure. It showed that the repository could not merely say it was ready; it had to carry inspectable proof.

## Evidence Added

The branch now carries metadata-only self-dogfood evidence under `.certamerge/evidence/`:

- test result evidence
- owner approval evidence
- dependency review evidence
- CAR verification evidence
- no-source-egress evidence
- risk surface classification evidence
- workflow validation evidence
- action contract validation evidence
- schema validation evidence
- compliance-safe-language evidence
- no-secret-leakage evidence
- link validation evidence

These files are not source-code payloads. They are small proof references that let the deterministic policy engine decide whether required evidence exists.

## Policy Surfaces

The self-dogfood policy covers:

- `community/cli/**`
- `community/cli/certamerge/car.py`
- `community/cli/certamerge/verifier.py`
- `community/cli/certamerge/schema.py`
- `community/tests/**`
- `community/github-action/**`
- `.github/workflows/**`
- `specs/**`
- `samples/**`
- public README, security, contributing, community docs, demo docs, release docs, research docs
- packaging and license files

It intentionally excludes enterprise-only paths and keeps the public repository focused on the community alpha surface.

## Current Self-Gate Result

Command:

```powershell
python -m certamerge gate --repo . --policy .certamerge.yml --output .tmp\dogfood-pr.car.json
```

Observed output:

```text

Policy reason: All matched policy requirements are satisfied.

Accountable next action: repo-owner - Proceed with record.
```

## What This Proves

This proves that CertaMerge can evaluate its own public alpha repository with a deterministic local policy, distinguish missing proof from satisfied proof, emit a CAR, and verify that CAR without relying on an LLM or vendor service.

## What This Does Not Prove

This does not prove production security, compliance certification, non-repudiable approval, or enterprise readiness. Community alpha CARs are SHA-256 content-hash verified but not cryptographically signed.
