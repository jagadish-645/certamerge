# Self-Dogfood Fix Log

## Fix 1: Root Self-Dogfood Policy

Finding: the repository did not have a serious self-governing policy for its own public alpha changes.

Why it matters: without a repo-local policy, CertaMerge could not prove that it uses its own verdict and CAR grammar.

Fix: added `.certamerge.yml` in observe mode with path rules for CLI, CAR/verifier, GitHub Action/workflow, tests, specs, samples, docs, packaging, and license surfaces.

Proof: `python -m certamerge gate --repo . --policy .certamerge.yml --output .tmp\strict-self.car.json` returned `OBSERVE_ONLY_WOULD_ALLOW`.

Remaining limitation: observe mode does not block merges by itself.

## Fix 2: Metadata Evidence Vocabulary

Finding: the policy needed evidence types beyond the initial test, CI, owner approval, artifact, security doc, and dependency reference set.

Why it matters: high-trust product self-governance needs proof for CAR verification, no-source-egress, risk classification, workflow validation, action contract validation, schema validation, compliance-safe language, secret leakage, and link validation.

Fix: expanded evidence normalization and signal detection in `community/cli/certamerge/evidence.py`.

Proof: `community/tests/test_controlled_alpha_contracts.py` now pins the alias map and metadata evidence file recognition.

Remaining limitation: metadata evidence presence is local and deterministic, but the community alpha does not yet cryptographically attest that every command was executed by an external runner.

## Fix 3: Self-Dogfood Evidence Files

Finding: the stricter policy needed repository-local proof artifacts.

Why it matters: policy requirements should map to concrete evidence, not narrative claims.

Fix: added metadata-only files under `.certamerge/evidence/`.

Proof: self-gate reports no missing proof under the current policy.

Remaining limitation: these files are community-alpha proof references, not signed attestations.

## Fix 4: GitHub Proof Gate Workflow

Finding: pull requests did not automatically run CertaMerge on CertaMerge.

Why it matters: a future contributor should see a CertaMerge verdict, policy reason, missing proof, accountable action, and CAR artifact without running the CLI manually first.

Fix: added `.github/workflows/certamerge-proof-gate.yml`.

Proof: static tests validate that the workflow uses the local action, points at `.certamerge.yml`, writes `.tmp/certamerge-pr.car.json`, verifies the CAR, and uploads `certamerge-pr-car`.

Remaining limitation: live GitHub runner validation happens after the draft PR is opened.

## Fix 5: PR And Issue Templates

Finding: public contribution surfaces did not ask for CertaMerge proof or enough safe reproduction context.

Why it matters: CertaMerge should make contributors speak in proof language without asking them to paste sensitive material.

Fix: updated `.github/PULL_REQUEST_TEMPLATE.md` and added issue templates for bug reports, alpha feedback, proof gap examples, and evidence adapter requests.

Proof: release candidate tests assert the files exist, avoid local path/token shapes, and include CertaMerge proof language.

Remaining limitation: templates guide contributors; they do not enforce correctness without CI and reviewer follow-through.

## Fix 6: CLI Missing-CAR Handling

Finding: verifier commands did not present a clean user-facing error for missing CAR paths.

Why it matters: a proof workflow will fail in normal ways; failed proof should become a clear next action, not an exception dump.

Fix: `load_car` now raises a user-facing `ValueError` for unreadable CAR files, and `verify-car`/`explain-car` return nonzero with a clear message.

Proof: tests assert missing files return exit code `1` and include `CAR file could not be read`.

Remaining limitation: this improves local CLI ergonomics only; richer remediation hints can come later.

## Fix 7: PR #2 Separation

Finding: PR #2 was a draft README rewrite that failed public README contract tests.

Why it matters: a polished public README can still remove product-critical proof wording.

Fix: recorded the PR #2 review separately and kept this dogfood branch independent.

Proof: `docs/release/PR_2_REVIEW_AND_NEXT_ACTION_REPORT.md` records the failed check and missing public contract phrases.

Remaining limitation: PR #2 still needs a separate fix before it should leave draft.
